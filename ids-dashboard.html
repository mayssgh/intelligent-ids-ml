Why SMOTE? — Class Imbalance Problem
The Problem: CICIDS2017 is heavily imbalanced. Heartbleed has only ~11 samples vs ~55k Benign flows. Without balancing, a model can achieve high accuracy simply by predicting "BENIGN" every time — this is a fake high accuracy.
SMOTE Solution: Synthetic Minority Over-sampling Technique generates new synthetic samples for minority classes by interpolating between existing samples. This gives the model equal exposure to all attack types during training.

Model Explainability — SHAP Analysis
Understanding why the Random Forest makes each prediction using SHAP values
SHAP (SHapley Additive exPlanations) — Computes the contribution of each feature to every individual prediction. A positive SHAP value pushes the prediction toward "Attack"; a negative value pushes toward "Benign". The summary plot shows feature importance across 200 test samples.
